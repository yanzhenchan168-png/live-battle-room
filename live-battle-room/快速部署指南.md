# 🎯 5 分钟快速部署指南

让正常用户访问你的应用，只需要 3 个步骤！

---

## 第 1 步：推送到 GitHub（2 分钟）

### 1.1 创建 GitHub 仓库
1. 访问 [github.com/new](https://github.com/new)
2. 创建新仓库：`live-battle-room`
3. 不要初始化 README（因为我们已经有了）

### 1.2 推送代码

```bash
# 进入项目目录
cd /workspace/projects/live-battle-room

# 初始化 Git
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 添加远程仓库
git remote add origin https://github.com/你的用户名/live-battle-room.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

**替换** `你的用户名` 为你的 GitHub 用户名。

---

## 第 2 步：部署到 Vercel（2 分钟）

### 2.1 访问 Vercel
打开 [vercel.com/new](https://vercel.com/new)

### 2.2 导入项目
1. 使用 GitHub 账号登录 Vercel
2. 点击 **"Add New..."** → **"Project"**
3. 找到 `live-battle-room` 仓库，点击 **"Import"**

### 2.3 配置项目
1. **Framework Preset**: 自动检测为 Next.js ✓
2. **Root Directory**: 保持默认 `./` ✓
3. 点击 **"Create"**

**环境变量配置：**

创建项目后，在项目设置中添加环境变量：

1. 点击项目 → **Settings** → **Environment Variables**
2. 添加以下 3 个变量：

```
NEXT_PUBLIC_COZE_BOT_ID
值: 7607002201290211382

COZE_TOKEN
值: pat_dtlvGNjEEDXRaEjukLBpCq3gS4H5uPGAC47xEI14H5xwfdJwHlGDM1e5TEjUEfJ4

DINGTALK_WEBHOOK_URL
值: https://oapi.dingtalk.com/robot/send?access_token=aa451ea0ebb48a4bead46231acc24f2072f3a2f8af16aa4bd826359e6fe1c396
```

3. 点击 **"Save"**
4. 重新部署：在 Deployments 页面，点击最新部署右侧的 **⋮** → **"Redeploy"**

---

## 第 3 步：分享给用户（1 分钟）

### 3.1 获取应用地址

部署成功后，Vercel 会显示一个地址：

```
https://live-battle-room-xxx.vercel.app
```

### 3.2 分享给用户

直接将这个地址发送给用户即可！

**特点：**
- ✅ 有 HTTPS（绿色锁图标）
- ✅ 全球 CDN 加速
- ✅ 移动端友好
- ✅ 无需绕过安全警告

### 3.3 配置自定义域名（可选）

如果有自己的域名：

1. 在项目设置 → **Domains**
2. 添加域名：`live-battle.yourdomain.com`
3. 配置 DNS 记录（Vercel 会给出提示）

## 🆘 遇到问题？

### 问题 1：部署后 API 调用失败

**解决方案**：检查环境变量是否正确配置
- 项目设置 → Environment Variables
- 确认 3 个变量都已添加

### 问题 2：部署很慢

**解决方案**：正常现象，首次部署需要 3-5 分钟

### 问题 3：用户访问报错

**解决方案**：
1. 检查部署状态是否为 "Ready"
2. 查看 Function Logs 查看错误信息
