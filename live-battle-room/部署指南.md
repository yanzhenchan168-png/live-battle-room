## 📋 部署方式选择

### 推荐：Vercel（最简单，官方推荐）

**优点**：
- ✅ Next.js 官方推荐的部署平台
- ✅ 自动 HTTPS（免费 SSL 证书）
- ✅ 自动 CDN 加速
- ✅ 免费额度充足（100GB/月）
- ✅ 零配置部署
- ✅ 全球边缘节点

**适用场景**：
- 个人项目
- 中小型应用
- 快速上线

## 🎯 方式 1：Vercel 部署（推荐）

### 步骤 1：准备代码

1. 确保代码已提交到 Git 仓库

```bash
cd /workspace/projects/live-battle-room
git init
git add .
git commit -m "Initial commit"
```

2. 推送到 GitHub/GitLab/Bitbucket

```bash
# 如果使用 GitHub
git remote add origin https://github.com/你的用户名/live-battle-room.git
git branch -M main
git push -u origin main
```

### 步骤 2：在 Vercel 创建项目

1. 访问 [vercel.com](https://vercel.com) 并登录
2. 点击 **"New Project"**
3. 导入你的 Git 仓库
4. 配置项目信息：
   - **Framework Preset**: Next.js
   - **Root Directory**: `./`
   - **Build Command**: `pnpm build`
   - **Output Directory**: `.next`

### 步骤 3：配置环境变量

在 Vercel 项目设置中添加以下环境变量：

```env
NEXT_PUBLIC_COZE_BOT_ID=7607002201290211382
COZE_TOKEN=pat_dtlvGNjEEDXRaEjukLBpCq3gS4H5uPGAC47xEI14H5xwfdJwHlGDM1e5TEjUEfJ4
DINGTALK_WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=aa451ea0ebb48a4bead46231acc24f2072f3a2f8af16aa4bd826359e6fe1c396
```

**操作步骤**：
1. 在项目页面点击 **"Settings"** → **"Environment Variables"**
2. 点击 **"Add New"**
3. 分别添加上述三个环境变量
4. 点击 **"Save"**

### 步骤 4：部署

1. 点击 **"Deploy"** 按钮
2. 等待构建完成（约 2-5 分钟）
3. 部署成功后，你会获得一个类似这样的域名：
   ```
   https://live-battle-room-xxx.vercel.app
   ```

### 步骤 5：配置自定义域名（可选）

1. 在项目页面点击 **"Settings"** → **"Domains"**
2. 添加你的域名（例如：`live-battle.yourdomain.com`）
3. 按照提示配置 DNS 记录

## 🎯 方式 2：Netlify 部署

### 步骤 1：配置 Netlify

创建 `netlify.toml` 文件：

```toml
[build]
  command = "pnpm build"
  publish = ".next"

[[plugins]]
  package = "@netlify/plugin-nextjs"

[functions]
  node_bundler = "esbuild"
```

### 步骤 2：连接到 Netlify

1. 访问 [netlify.com](https://netlify.com)
2. 点击 **"Add new site"** → **"Import an existing project"**
3. 连接你的 Git 仓库
4. 配置环境变量（同 Vercel）

### 步骤 3：部署

点击 **"Deploy site"**，等待构建完成。

## 🎯 方式 3：Docker 部署（企业级）

### 步骤 1：创建 Dockerfile

```dockerfile
FROM node:22-alpine AS base

# 安装依赖
FROM base AS deps
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN npm install -g pnpm && pnpm install --frozen-lockfile

# 构建
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN pnpm build

# 运行
FROM base AS runner
WORKDIR /app
ENV NODE_ENV=production

COPY --from=builder /app/public ./public
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static

EXPOSE 3000

CMD ["node", "server.js"]
```

### 步骤 2：构建镜像

```bash
docker build -t live-battle-room .
```

### 步骤 3：运行容器

```bash
docker run -p 3000:3000 \
  -e NEXT_PUBLIC_COZE_BOT_ID=7607002201290211382 \
  -e COZE_TOKEN=pat_dtlvGNjEEDXRaEjukLBpCq3gS4H5uPGAC47xEI14H5xwfdJwHlGDM1e5TEjUEfJ4 \
  -e DINGTALK_WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=aa451ea0ebb48a4bead46231acc24f2072f3a2f8af16aa4bd826359e6fe1c396 \
  live-battle-room
```

## 🎯 方式 4：云服务器部署

### 步骤 1：购买云服务器

推荐：
- 阿里云 ECS
- 腾讯云 CVM
- 华为云 ECS
- AWS EC2

### 步骤 2：安装环境

```bash
# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 pnpm
npm install -g pnpm

# 安装 PM2（进程管理）
npm install -g pm2
```

### 步骤 3：部署应用

```bash
# 克隆代码
git clone https://github.com/你的用户名/live-battle-room.git
cd live-battle-room

# 安装依赖
pnpm install

# 配置环境变量
cat > .env.production << EOF
NEXT_PUBLIC_COZE_BOT_ID=7607002201290211382
COZE_TOKEN=pat_dtlvGNjEEDXRaEjukLBpCq3gS4H5uPGAC47xEI14H5xwfdJwHlGDM1e5TEjUEfJ4
DINGTALK_WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=aa451ea0ebb48a4bead46231acc24f2072f3a2f8af16aa4bd826359e6fe1c396
EOF

# 构建
pnpm build

# 使用 PM2 运行
pm2 start npm --name "live-battle-room" -- start
pm2 save
pm2 startup
```

### 步骤 4：配置 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name live-battle.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 步骤 5：配置 SSL（使用 Let's Encrypt）

```bash
# 安装 Certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d live-battle.yourdomain.com
```

## 🔐 环境变量安全

### ⚠️ 重要提示

1. **不要将敏感信息提交到 Git**
   ```bash
   # 添加到 .gitignore
   echo ".env.local" >> .gitignore
   echo ".env.production" >> .gitignore
   ```

2. **在部署平台配置环境变量**
   - 不要在代码中硬编码密钥
   - 使用部署平台的环境变量功能
   - 定期轮换 API Token

3. **使用密钥管理服务**
   - 生产环境推荐使用 AWS Secrets Manager
   - 或使用 HashiCorp Vault
